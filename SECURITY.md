# 网络安全说明与加固指南

## 项目已做的安全措施

1. **登录态 Cookie**
   - `httpOnly`：脚本无法读取，降低被恶意脚本盗取的风险。
   - `sameSite: "lax"`：降低跨站请求伪造（CSRF）风险。
   - 生产环境下 `secure: true`：仅在 HTTPS 下发送，防止在非加密网络中被窃听。

2. **安全响应头**（在 `next.config.ts` 中）
   - `X-Frame-Options: DENY`：防止页面被嵌入到其他网站（点击劫持）。
   - `X-Content-Type-Options: nosniff`：避免浏览器按错误类型解析响应。
   - `Referrer-Policy`：控制 Referer 泄露。
   - `Permissions-Policy`：限制摄像头、麦克风、定位等权限，本应用不需要这些能力。

3. **无 XSS 注入点**
   - 未使用 `dangerouslySetInnerHTML`，页面内容由 React 默认转义渲染，用户输入不会被当作 HTML/脚本执行。

4. **密码与鉴权**
   - 密码使用 bcrypt 哈希存储，不明文保存。
   - 会话使用 JWT（jose 库）签发与校验，接口按登录态和权限做校验。

## 您需要完成的加固步骤（上线前必做）

### 1. 全程使用 HTTPS（最重要）

- **必须让老师通过 HTTPS 访问**（例如 `https://你的域名.com`），不要用 `http://`。
- 部署到 Vercel / 腾讯云 / 其他云时，在控制台开启 HTTPS（一般会提供免费证书）。
- 若自建服务器，请配置 Nginx 等反向代理并启用 SSL 证书（如 Let’s Encrypt）。
- 只有 HTTPS + 生产环境，登录 Cookie 才会以 `secure` 方式发送，避免在公共 Wi‑Fi 等环境下被窃听或篡改。

### 2. 设置强 JWT 密钥

- 在**生产环境**的 `.env` 中设置 `JWT_SECRET`，且不要使用代码里的默认值。
- 使用随机、足够长的字符串（例如 32 位以上随机字符），且不要提交到 Git。
- 示例生成方式（在本地终端执行一次，把输出填到服务器环境变量）：
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```

### 3. 数据库与环境变量

- `DATABASE_URL` 等敏感信息只放在服务器环境变量或托管平台的环境配置中，不要写进代码或提交到代码库。
- 生产库与测试库分开，生产环境不要使用测试账号或弱密码。

### 4. 依赖安全

- 定期执行：
  ```bash
  npm audit
  ```
- 对报告的漏洞按提示修复（`npm audit fix` 或按建议升级依赖），避免已知漏洞被利用。

### 5. 部署与访问方式

- 若通过微信/企业微信等内置浏览器打开，请确保最终跳转的是 **HTTPS** 地址。
- 建议老师仅在可信网络下登录；在公共 Wi‑Fi 下也务必使用 HTTPS，避免中间人窃听。

## 总结

| 风险类型           | 说明与应对 |
|--------------------|------------|
| “病毒侵入手机”     | 本系统为网页应用，不会在手机上安装软件；正常使用不会因此中病毒。 |
| 账号被盗 / 冒用    | 依赖 HTTPS + 强 JWT 密钥 + 安全 Cookie，并提醒老师不与他人共享密码。 |
| 数据被窃听或篡改   | 全程 HTTPS + 生产环境启用 `secure` Cookie。 |
| 依赖漏洞           | 定期 `npm audit` 并修复。 |

按上述步骤配置后，老师用手机通过 **HTTPS** 访问本系统，并配合强密钥与依赖维护，可以显著提升系统与用户数据的安全性。若后续有公网暴露的管理后台或敏感接口，可再考虑限 IP、限频、二次验证等进一步加固。
